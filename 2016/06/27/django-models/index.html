django-models

Posted on 2016-06-27   |  

django.db.models.Model

  • 每个模型都是django.db.modles.Model的子类
  • 模型的每个属性都是数据库中的一个字段

Field

模型中的每个字段都是Field子类的某个实例。Django根据字段类的类型确定一下信息：

  • 数据库当中的列类型（比如：INTERGER, VARCHAR）。
  • 渲染表单是使用的默认HTML部件（例如，<input type="text">,<select>)。
  • 最低限度的验证需求，它被用在Django管理站点和自动生成的表单中。

Field选项

  • null=True，数据库中使用NULL作为存储控制
  • blank=True， 表示允许该段不填，blank=False则表示该字段必须填写
  • choices， 第一个数字表示数据库中的值，第二个表示管理界面中显示的值。

自增主键字段

默认情况下，Django会给每个模型添加下面字段：
id = models.AutoField(primary_key = True)
指定特定的字段为主键，只要在某个字段上加上primary_key = True

字段自述名

这个例子自述名是“person's fisrt name":
first_name = models.CharField(“person's first name”, max_length=30)
或者指定verbose_name，verbose_name="the related poll"

多对多关系

例如，有这样一个应用，它记录音乐家所属的音乐小组。我们可以用一个ManyToManyField 表示小组和成员之间的多对多关系。但是，有时你可能想知道更多成员关系的细节，比如成员是何时加入小组的。
对于这些情况，Django 允许你指定一个中介模型来定义多对多关系。 你可以将其他字段放在中介模型里面。源模型的ManyToManyField字段将使用through参数指向中介模型。

中介模型有一些限制：

  • 中介模型必须有且仅有一个外键到原模型（Group），或者你必须使用ManyToManyField.through_fields显示指定Django应该在关系中使用的外键。
  • 对于通过中介模型与自己进行多对多关联的模型，允许存在到同一个模型的两个外键，但它们将被作为多对多关联关系的两个（不同的）方面。如果有超过 两个外键，同样你必须像上面一样指定through_fields，否则将引发一个验证错误。
  • 使用中介模型定义与自身的多对多关系时，你必须设置 symmetrical=False。（参考）

Meta(元)

元数据是“任何不适字段的数据”，比如排序选项(ordering)，数据库表名(db_table)或者人类可读的单复数名称(verbose_name和verbose_name_plural)。

重写预定义的模型方法

修改save()，delete()方法。

模型继承

Django中有3个风格的继承

  1. 如果只希望使用父类的一些信息，而不想每个子模型中都敲一遍，所以使用抽象基类。
  2. 如果你继承一个已经存在的模型且让每个模型都具有自己的数据库表，那么应该使用多表继承。
  3. 如果你想改变一个模块python级别的行为，而不是修改模型的字段，则使用代理模型。

抽象基类

当你想将一些共有信息放进其它一些model的时候，抽象化类是十分有用的。你编写完基类之后，在 Meta类中设置 abstract=True，这个模型就不会被用来创建任何数据表。

继承时，子类的abstract=False，除非手动处理abstract=True。

多表继承

继承的子类不会继承父类的Meta，除了ordering或者get_lastest_by等
继承默认隐式创建的是OneToOneField，当显式继承ManyToManyField时，需要指定related_name

代理继承

MyPerson类和它的父类 Person 操作同一个数据表。特别的是，Person 的任何实例也可以通过 MyPerson访问，反之亦然。
你也可以使用代理 model 给 model 定义不同的默认排序设置。 你可能并不想每次都给Person模型排序，但是使用代理的时候总是按照last_name属性排序。
